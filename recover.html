{help users recognize, diagnose, and recover from errors}

“Error messages should be expressed in plain language (no codes), precisely indicate the problem, and constructively suggest a solution.”

Unhelpful Error Messages:

UL is adept at producing error messages but not always ones that are easily understood by the average user. Furthermore, these cryptic error messages do not contain possible solutions- meaning these messages don’t do the user any good! ULink, the university’s online services, does provide a good error message when a user enters an incorrect username and/or password. The site’s search bar is adept at catching spelling errors and predicting what the user intended to find.

Overall:
The website is not completely void of error messages, but some of them do not do a good job of guiding the user to a possible solution. Users would be less frustrated upon encountering a problem if the site provided a clearer explanation of the problem, FAQ page, and possible ways to mend the issue.
