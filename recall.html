{recognition rather than recall}

“Minimize the user's memory load by making objects, actions, and options visible. The user should not have to remember information from one part of the dialogue to another. Instructions for use of the system should be visible or easily retrievable whenever appropriate.”

Navigation that is not User-Friendly:

This site is extensively labeled so that the user almost always knows where they are, where it is possible to go, but not exactly how they got there. Lack of breadcrumbs or any other kind of navigation aid forces users to rely on their memory to remember how to get to a particular page. The website does show a commendable attempt of placing headers and links and using non-technical language to inform users of what information they can expect to acquire. Unfortunately, the multiple headers and wordy text make the site hard to navigate.

Overall:
The UL site’s attempt to provide “more” is an unfortunate example of when more is detrimental. Lack of navigational tools make recognition rather difficult and emphasize recall over recognition.
